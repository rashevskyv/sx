    Moving from SX to Atmosphere

    Why switch to Atmosphere?

    In October 2020 two Xecuter Team leaders were arrested - a campaign engaged in the development and support of SXOS, as well as the production of chips for modifying set-top boxes with a closed vulnerability.

    On December 1, 2020, the Switch system software update under version 11.0.0 is released and after 5 days the SXOS team released a firmware supporting this update. Everything seems to be going well and despite the arrests, the XTs are still working.

    However, at about the same time, all Chinese sites that sold chips unanimously began to say that they had run out of chips and the prospects for the production of new batches were very dim. Rumors also began to spread that all chip manufacturing factories were closed and products had been confiscated. The staff was dispersed, and the ban on the sale of chips was launched from almost the top of the Chinese Communist Party. Which is not so hard to believe if you remember that the most powerful corporation Tencent is the official distributor of Nintendo products in China.

    The last straw confirming that evil tongues are probably right was the release of firmware 11.0.1, which at the time of this writing is still not working on SXOS.

    It should be noted that the 11.0.1 update was minor and Atmosphere was already working with it by the evening of the same day.

    At the beginning of 2021, Atmosphere 0.17.0 was released, which already fully supported work on chip set-top boxes, so now you don’t have to wait for new versions of SXOS - it’s just no longer necessary.

    Moving from SX to Atmosphere

    Choose the option that suits you and follow the instructions.

    Fusée Gelée flashing attachment or Caffeine flashing
    1. Insert the memory card of the set-top box into the PC
    2. Install kefir Сборник, состоящий из выбранного кастома, необходимых программ и скриптов, которые все это установят правильным образом. Состав кефира и инструкцию по установке можно посмотреть в его репозитории by choosing “Atmosphere” as an option (atmo.zip for manual installation)
    3. Wait for the installation to complete, then return the memory card to the console
    4. Turn on the console
    5. If you had EmuNAND, on the kefir screensaverpress the volume down buttonto get into hekate
    6. Go to “emuMMC”
    7. Click “Migrate emuMMC”
    8. Select “Emunand” -> “Continue”
    9. After finishing the migration, click “Change emuMMC” and on the right side of the screen select “emuMMC/EF00”
    10. Click “OK”
      • Now, when the STB starts up, it will automatically launch emuMMC.
      • To get into the official firmware, run hekate -> Launch -> Stock

    Preparatory work

    Until we start the transition, we need to make sure that SysNAND and EmuNAND are running the latest version of the system software. Otherwise, when migrating EmuNAND to Atmosphere, you will not be able to run it.

    Backing up saves

    1. Back up the save files according to this instruction

    Upgrading SysNAND to the latest system software

    1. Turn on the console while holding the volume up button to access the SX OS bootloader menu
    2. Go to “Options” -> “SX Core/SX Lite” -> “Genuine boot” -> “Continue” to run SysNAND
    3. Wait for the download of the official firmware to finish
    4. Go to “System Preferences” -> “System”, click on “System Update”
      • Wait for the firmware update, then turn off the console
      • If your set-top box already has the latest version of system software, just turn off the console
      • Yes, it is safe and will not result in a ban

    Updating EmuNAND to the latest system software

    1. Update EmuNAND to 11.0.1 by this instruction
    Start firmware
    1. Insert the memory card of the set-top box into the PC
    2. Install kefir Сборник, состоящий из выбранного кастома, необходимых программ и скриптов, которые все это установят правильным образом. Состав кефира и инструкцию по установке можно посмотреть в его репозитории by choosing “Atmosphere for SX Core/Light” as an option (modcip.zip for manual installation)
    3. Wait for the installation to complete, then return the memory card to the console
    4. Turn on the console
      • SysNAND should load.
      • If you encounter any errors, see the “Problems and Solutions” section at the bottom of this page
    Transferring EmuNAND from SX OS to Atmosphere
    1. Turn off your console
    2. Turn on the console while holding the volume up button to boot into hekate
    3. Go to “emuMMC”
    4. Click “Migrate emuMMC”
    5. Select “Emunand -> “Continue”
    6. After finishing the migration, click “Change emuMMC” and on the right side of the screen select “emuMMC/EF00”
    7. Click “OK”
    8. Go to “Tools” -> “Backup eMMC” -> “eMMC BOOT0 & BOOT1”
    9. Press “Close”, “Close”, go to “Home” and press “Power Off”, confirm shutdown
    10. Insert the memory card of the set-top box into the PC
    11. Go to the folder sd: \ backup \% emmc_id%, where% emmc_id%is a unique number for each separate STB.
    12. Copy BOOT0 and BOO1 from this folder to the folder emuMMC \ ED00 \ eMMC with replacement
      • Now, when the STB starts up, it will automatically launch emuMMC.
      • To get into the official firmware, runhekate -> “Launch” -> “Stock”
    Creation of EmuNAND via hekate

    If you didn’t have EmuNAND, it is highly recommended to create one. This is done primarily to avoid unpleasant consequences for SysNAND in case of any problems, so do EmuNAND even if you don’t plan to use a license or online!

    1. Turn off your console
    2. Turn on the console while holding the volume up button to boot into hekate
    3. Go to “emuMMC”
    4. Click “Create emuMMC” -> “SD File”
      • To create emuMMC you need to have at least 32GB of free space on the card! If you do not have that much, take another card, or temporarily transfer the Nintendo folder to your PC
    5. After finishing emuMMC creation, click “OK”
      • Now, when the STB starts up, it will automatically launch emuMMC.
      • To get into the official firmware, runhekate -> “Launch” -> “Stock”
    Backup firmware and keys
    1. Turn off your console
    2. Insert the memory card of the set-top box into the PC
    3. Copy the emuMMC folder from the root of your memory card to your PC. This is your emunand. It is identical to the backup, so there is no point in creating a backup separately
    4. Return the memory card to the console
    5. Turn on the console while holding the volume up button to boot into hekate
    6. Go to Payloads and select Lockpick_RCM.bin
    7. Press the (Power) button, being on the first line in the program for dumping keys
    8. Press the (Power) button again to go to the main menu of the program.
    9. Use the volume buttons to move the cursor to the Power Off item and press the (Power) button to turn off the set-top box.
    10. Insert the memory card of the set-top box into the PC
    11. Go to the folder sd:/switch and copy the files prod.keys and partialaes.keys to the same folder where you copied your backup

    Place the backup in a safe place, preferably several different ones. And into the cloud. And on a USB flash drive. Don’t lose it!

    Problems and solutions

    Black screen when starting SysNAND
    1. Turn off the set-top box by holding the power button for 30 seconds
    2. Insert the memory card of the set-top box into the PC
    3. Download SX_OS 3.1.0
    4. Place boot.dat from the archive SX_OS_v3.1.0_BETA_by_ [Team-Xecuter] .zip into the root of the memory card with replacement
    5. Turn on the console while holding the volume up button to access the SX OS bootloader menu
    6. Go to “Options” -> “SX Core/SX Lite”, then selectCleanup -> Continue
    7. Press “Power Off” to turn off the console
    8. Insert the memory card of the set-top box into the PC
    9. Install kefir Сборник, состоящий из выбранного кастома, необходимых программ и скриптов, которые все это установят правильным образом. Состав кефира и инструкцию по установке можно посмотреть в его репозитории by choosing “Atmosphere for SX Core/Light” as an option (modcip.zip)
    10. Wait for the installation to complete, then return the memory card to the console
    11. Turn on the console
      • SysNAND should load.
    Black screen when starting EmuNAND/Errors when starting EmuNAND related to boot

    Try starting SysNAND first and only follow the instructions below if SysNAND starts! If not, fix this problem first and then go back to this tutorial!

    1. Turn on the console
    2. On the kefir screensaverpress the volume down buttonto get into hekate
    3. Go to “Tools” -> “Backup eMMC” -> “eMMC BOOT0 & BOOT1”
    4. Press “Close”, “Close”, go to “Home” and press “Power Off”, confirm shutdown
    5. Insert the memory card of the set-top box into the PC
    6. Go to the folder sd: \ backup \% emmc_id%, where %emmc_id% is a unique number for each separate STB.
    7. Copy BOOT0 and BOO1 from this folder to the folder emuMMC \ ED00 \ eMMC with replacement
    8. Try running emunand. If it doesn’t work, you will have to create a new one.
    Error "Pkg1 decryption failed! Is BEK Missing?"

    Update kefir Сборник, состоящий из выбранного кастома, необходимых программ и скриптов, которые все это установят правильным образом. Состав кефира и инструкцию по установке можно посмотреть в его репозитории !
